# Claude Review Skill

## 목적
코드 변경 사항을 현재 세션과 독립된 Claude CLI로 교차 검토하고, 결과 또는 실패 사유를 반드시 보고한다.

## 규칙

### 사전 확인
1. 리뷰 호출 전에 아래 명령으로 Claude CLI 동작 여부를 확인한다.
   ```powershell
   & 'C:\Users\nowonbun\AppData\Roaming\npm\claude.cmd' --version
   & 'C:\Users\nowonbun\AppData\Roaming\npm\claude.cmd' -p "Return exactly: OK"
   ```
2. `OK`가 반환되면 정상. 응답이 없거나 에러면 아래 실패 처리 규칙을 따른다.

### 호출 방식
3. `-p` 인자에 diff 내용이나 파일 전체를 직접 붙여넣지 않는다. Windows 명령줄 길이 제한(약 8191자)에 걸려 `The command line is too long.` 에러가 발생한다.
4. 짧은 지시문만 `-p` 인자로 전달하고, 파일은 Claude가 작업 디렉터리에서 직접 읽게 한다.
   ```powershell
   # 올바른 패턴
   Set-Location D:\work\<저장소>
   & 'C:\Users\nowonbun\AppData\Roaming\npm\claude.cmd' -p "<파일명>의 현재 변경 사항을 리뷰해라."

   # 잘못된 패턴 (길이 초과 에러 발생)
   & 'C:\Users\nowonbun\AppData\Roaming\npm\claude.cmd' -p "<diff 전체 또는 파일 전체 내용>"
   ```
5. `claude.ps1` 대신 `claude.cmd`를 우선 사용한다.
6. 샌드박스 실행을 시도하지 말고 권한 상승으로 바로 실행한다.

### 보고 형식
7. 리뷰가 성공한 경우 아래 완료 보고 형식을 사용한다.
8. 리뷰가 실패한 경우 아래 실패 보고 형식을 사용한다.
9. 어떤 경우에도 리뷰 단계를 무보고로 생략하지 않는다.

### 실패 처리
10. `The command line is too long.` 에러 → 지시문을 더 짧게 줄이거나 파일명만 명시해 재시도한다.
11. 60초 이상 무응답 → 지시문을 조사 단계와 리뷰 단계로 분할해 재시도한다.
12. 재시도 후에도 실패하면 리뷰를 생략하고, 아래 실패 보고 형식으로 사유와 대체 검증 결과를 명시한다.

## 리뷰 요청 템플릿

Claude CLI에 전달하는 `-p` 지시문은 아래 구조를 따른다.

```text
[Role]
너는 코드 리뷰 담당이다.
현재 세션과 독립된 시점에서 변경 사항의 정합성·부작용·롤백 가능성을 판단한다.

[Task]
<리뷰 대상 파일명 또는 변경 요점 한 줄 요약>

[Output Format]
1) 결론 (1~2줄)
2) 지적 사항 (논점·근거·영향 범위)
3) 채택 권고 / 수정 필요 여부
※ 근거 없는 단정, 아부성 표현 금지
```

## 리뷰 완료 보고 형식

Claude 리뷰가 성공하면 아래 형식으로 보고한다.

```text
[Claude 리뷰 완료]
- 검토 대상: <파일명 또는 변경 범위>
- 지적 논점:
  - <논점 1>
  - <논점 2>  ← 없으면 "없음"
- 채택 결과: 전체 채택 | 일부 채택 | 기각
- 채택/기각 이유: <이유>
- 반영 여부: 반영 완료 | 추가 수정 예정 | 기각
```

## 리뷰 실패 보고 형식

CLI 호출이 실패하거나 재시도 후에도 불가능한 경우 아래 형식으로 보고한다.

```text
[Claude 리뷰 실패]
- 실패 원인: 명령줄 길이 초과 | CLI 미응답 | 권한 오류 | 기타(<내용>)
- 시도 내용: <전달한 지시문 요약>
- 재시도 횟수: <N회>
- 대체 검증 방법: <수동 확인 내용 또는 "없음">
- 대체 검증 결과: <결과 또는 "생략">
```
