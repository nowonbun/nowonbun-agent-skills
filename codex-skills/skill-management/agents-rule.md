# Must

## Scope
- 이 문서는 각 폴더의 `AGENTS.md`를 작성하거나 개정할 때 적용한다.
- 목적은 폴더별 역할, 로컬 핵심 규칙, 스킬/MCP 운용 방식을 명확히 하는 것이다.

## Out of Scope
- 전역 원칙은 `global_instructions.md`에서 다룬다.
- 개별 스킬 문서 작성 규칙은 `skill-create-rule.md`를 따른다.
- 일반 설명 문서나 이력 문서에는 이 문서를 직접 적용하지 않는다.

## Structure
1. `AGENTS.md`는 해당 폴더의 역할을 먼저 설명한다.
2. 해당 폴더에서 우선 참조할 문서와 규칙을 적는다.
3. 어떤 스킬을 어떤 상황에서 우선 적용할지 적는다.
4. MCP를 사용한다면 사용 목적, 확인할 파라미터/반환 형식, 호출 전 주의사항을 적는다.
5. 폴더 전용 예외 규칙이 있으면 전역 규칙과 충돌하지 않게 범위를 명시한다.
6. 구조적 관계, 절차 흐름, 상태 전이, 승인 흐름처럼 텍스트만으로 오해될 수 있는 내용은 PlantUML(`puml`)로 표현한다.

## Rule Writing
1. 문서는 폴더 범위의 운영 규칙만 다룬다.
2. 하위 스킬 전체를 다시 설명하지 말고 우선 적용 기준만 적는다.
3. MCP 설명은 사용 맥락 중심으로 적고, 상세 스펙 복제는 피한다.
4. 폴더 역할과 맞지 않는 규칙은 넣지 않는다.

## Source of Truth
1. 해당 폴더의 로컬 운영 기준의 단일 원본은 그 폴더의 `AGENTS.md`다.
2. 전역 원칙의 단일 원본은 `global_instructions.md`다.
3. 개별 스킬 작성 규칙의 단일 원본은 `skill-create-rule.md`다.

# Do NOT

## Wrong Scope
- 전역 규칙을 `AGENTS.md`에서 새로 정의하지 않는다.
- 개별 스킬 전체 내용을 `AGENTS.md`에 복제하지 않는다.
- 폴더와 무관한 MCP 설명을 넣지 않는다.

## Ambiguity
- 어떤 스킬을 언제 쓰는지 기준 없이 나열하지 않는다.
- MCP 사용 시 확인 항목을 `적절히`, `필요시` 같은 표현으로 흐리지 않는다.

# Flow

## Authoring
1. 폴더의 역할과 책임 범위를 정의한다.
2. 전역 규칙과 로컬 예외를 분리한다.
3. 우선 적용 문서와 스킬을 정리한다.
4. MCP가 있으면 사용 맥락과 주의사항을 정리한다.
5. 완료 후 `Verification`과 필요 시 `Monitoring`을 작성한다.

# Definition of Done

## Verification
- `AGENTS.md`가 폴더 역할을 먼저 설명한다.
- 로컬 핵심 규칙과 우선 문서가 명시돼 있다.
- 어떤 스킬을 언제 우선 적용할지 기준이 있다.
- MCP 사용 여부와 사용 시 확인 항목이 있다.
- 전역 규칙과 로컬 예외의 경계가 분명하다.

## Monitoring
- 폴더 구조 변경, 사용 스킬 변경, MCP 연결 방식 변경이 생기면 `AGENTS.md`를 다시 검토한다.
- `AGENTS.md`의 안내와 실제 폴더 운영 방식이 반복해서 어긋나면 로컬 규칙을 갱신한다.
