# Stock Analysis Skill

# Must

## Scope
- 이 문서는 투자 검토, 거래 검토 또는 학습 목적의 비교 분석을 위해 주식을 분석할 때 반드시 적용해야 합니다.
- 이 문서는 증거 기반 주식 분석 결과를 도출하기 위한 구조화된 도구로 사용해야 하며, 수익을 보장하거나 확실한 결과를 제시하는 용도로 사용해서는 안 됩니다.

## Source of Truth
- 이 문서는 `./SKILL.md`에 있는 주식 분석 구조, 증거 제시 방식 및 위험 보고 규칙에 대한 유일한 기준 문서입니다. 거버넌스 계층, 신속 처리 결정 또는 데이터 검색 도구 선택은 이 문서의 기준이 아닙니다.
- 주식 분석 요청이 더 광범위한 주제 연구로 확장될 경우, 사실과 해석을 구분하는 데 있어 `../action-management_research/SKILL.md`가 유일한 기준 문서입니다. 이는 주제 간 증거 구분이 이 주식별 구조 규칙이 아닌 `../action-management_research/SKILL.md`에 속하기 때문입니다.

- Stock MCP 도구 선택, 필수 매개변수 및 스캐너 검색 규칙은 `../tool-usage-management_stock-mcp/SKILL.md`를 단일 기준 문서로 참조해야 합니다.

## Analysis Setup Rules
- 결론을 제시하기 전에 분석 목적을 투자 검토, 거래 검토 또는 학습으로 명확히 정의해야 합니다.
- 결론을 제시하기 전에 분석 기간을 단기, 중기 또는 장기로 명확히 정의해야 합니다.
- 증거가 불완전하거나, 지연되었거나, 가정에 기반한 경우 불확실성과 분석의 한계를 명시해야 합니다.
- 요청에서 확실성 또는 고정 수익을 요구하는 경우, '수익 보장'이라는 틀을 사용해서는 안 됩니다.

## Evidence and Method Rules
- 숫자, 비율 또는 지표를 제시할 때는 출처 또는 계산 근거를 반드시 명시해야 합니다.
- 하나의 답변에 기술적 분석과 기본적 분석이 모두 포함될 경우, 두 분석을 구분하여 제시해야 합니다.
- 근거가 없는 가정은 `unverified`으로 표시해야 합니다.
- 사용자의 비현실적인 판단에 대해서는 직접적으로 이의를 제기하고, 더 안전한 대안이나 더 좁은 해석을 제시해야 합니다.

- 분석에 `stock_data`, `list_predict_dates`, `predict_rows` 또는 `scanner_rows`가 필요하다고 판단한 경우, 왜 그 도구가 필요한지와 어떤 데이터 공백을 메우는지를 먼저 설명해야 합니다.
- `stock_data`, `list_predict_dates`, `predict_rows`의 실제 선택 기준과 필수 매개변수는 이 문서에서 중복 정의하지 말고 `../tool-usage-management_stock-mcp/SKILL.md`를 참조해야 합니다.

## Minimum Analysis Coverage Rules
- 추세 관련 증거가 있는 경우 가격 추세 또는 수익률 추세를 요약해야 합니다.
- 시장 활동 관련 증거가 있는 경우 거래량 추이 또는 변동성 추이를 설명해야 합니다.
- 비교 관련 증거가 있는 경우 시장 흐름, 업종 흐름 또는 상대적 강도 맥락을 설명해야 합니다.
- 주요 위험 요소를 별도의 항목으로 나열하고, 관련 증거가 있는 경우 거시적 위험, 산업 위험 또는 기업별 위험을 포함해야 합니다.

- 후보군 선별이 먼저 필요한 분석에서는 필요한 경우 UpperBand 상단 스캐너 검색, 가격 상한, 거래대금 하한 같은 사전 필터 조사 가능성을 먼저 검토해야 합니다.
- 사용자가 "어떤 종목을 볼지" 또는 "조건에 맞는 후보군"을 먼저 요구하는 경우, 개별 종목 심화 분석 전에 `../tool-usage-management_stock-mcp/SKILL.md`의 스캐너 검색 규칙을 참조하여 후보군을 먼저 정리해야 합니다.
- 사용자가 "추천", "뭐 살까", "후보군", "선별"처럼 복수 종목 추천을 요청하는 경우, 개별 종목 심화 분석 전에 MCP로 후보군을 먼저 정리해야 합니다.
- 시장 조사 근거가 있는 경우 지수 흐름, 주도 업종 또는 테마, 확인 가능한 매출 증가·실적 방향성과 같은 기본적 맥락을 구분해서 요약해야 합니다.
- 매출 증가, 실적 개선, 테마 존재 여부를 근거로 제시하는 경우 실제 파일, MCP 응답, 공식 문서 또는 계산 근거가 있어야 하며, 근거가 부족하면 `unverified`로 표시해야 합니다.
- 추천 종목을 보수형, 중립형, 공격형처럼 분류하는 경우 변동성, 유동성, 추세 안정성, 불확실성 수준 중 확인 가능한 증거를 함께 제시해야 합니다.


## Example Analysis Procedure Rules
- 후보군 선별이 먼저 필요한 요청에서는 조건 검색 또는 스캐너 조사 여부를 먼저 판단하고, 후보군이 정리된 뒤 개별 종목 심화 분석으로 넘어가야 합니다.
- 단일 종목 심화 분석 요청에서는 가격 이력, 거래량 흐름, 변동성, 비교 대상, 주요 위험 요소 순으로 증거를 정리해야 합니다.
- 예측 데이터를 포함한 해석이 필요한 경우에는 사용 가능한 예측 기준일 확인과 예측 행 조회가 필요한지 먼저 설명하고, 세부 도구 선택과 매개변수는 `../tool-usage-management_stock-mcp/SKILL.md`를 참조해야 합니다.
- 답변 작성 예시는 `결론 -> 기술적/기본적/맥락적 근거 -> 주요 위험 -> 다음 단계` 순서를 따라야 합니다.
- 예시 1: "조건에 맞는 일본 종목을 먼저 찾고 그중 상위 3개를 분석" 요청에서는 후보군 선별, 후보군 요약, 개별 종목 심화 분석 순으로 진행해야 합니다.
- 예시 2: "특정 종목 하나를 분석" 요청에서는 후보군 탐색을 생략하고 해당 종목의 시계열 증거와 위험 검토를 우선 정리해야 합니다.
- 예시 3: "예측 데이터까지 같이 보고 싶다"는 요청에서는 예측 기준일 확인 필요성, 예측 데이터의 한계, 현 시점 가격 흐름과의 분리 보고 여부를 먼저 명시해야 합니다.
- 예시 4: "후보군 먼저 뽑고 시장조사까지 해서 추천해줘" 요청에서는 후보군 정리, 시장 조사 요약, 추천 종목 선별, 보수형/중립형/공격형 분류, 주요 위험 순으로 진행해야 합니다.

## Output Rules
- 결론을 먼저 제시해야 합니다.
- 뒷받침하는 근거는 글머리 기호를 사용하거나 명확하게 구분된 섹션으로 제시해야 합니다.
- 복수 종목 추천 보고서는 가능한 경우 `결론 -> 후보군 선별 근거 -> 시장 조사 -> 추천 종목 및 선별 이유 -> 위험 성향별 분류 -> 주요 위험 -> 다음 단계` 순서를 따라야 합니다.
- 추가 데이터 확인, 위험 검토 또는 시나리오 비교와 같은 실질적인 다음 단계로 마무리해야 합니다.

# Must NOT

- 주식 분석을 확실한 수익 보장 지표로 제시해서는 안 됩니다.
- 기술적 분석과 기본적 분석을 혼합하여 사용할 경우, 사용 방법을 명확히 밝혀야 합니다.
- 수치적 주장을 제시할 경우, 출처나 계산 근거를 제시해야 합니다.
- 불확실성이나 하락 위험을 확정적인 표현 뒤에 숨겨서는 안 됩니다.

# Flow

1. 요청된 주식 분석의 목적과 기간을 정의합니다.
2. 추천 또는 후보군 요청인 경우 MCP를 통해 후보군을 먼저 정리하고, 단일 종목 요청인 경우 해당 종목 증거 수집으로 바로 들어갑니다.
3. 증거를 수집하고 기술적, 펀더멘털적, 맥락적 신호를 구분합니다.
4. 시장 조사 근거가 있는 경우 지수, 업종, 테마, 기본적 맥락을 별도 섹션으로 요약합니다.
5. 추세, 거래량 또는 변동성, 비교 대상, 주요 위험 요소를 요약하고 필요한 경우 위험 성향별 분류를 정리합니다.
6. 결론을 먼저 제시하고 증거에 근거한 추론을 덧붙입니다.
7. 실질적인 다음 단계 또는 누락된 데이터 확인으로 마무리합니다.

# Definition of Done

## Verification
- 분석 목적과 분석 기간을 결론을 도출하기 전에 명확히 정의합니다.
- 기술적 분석과 기본적 분석을 모두 사용할 경우, 두 분석 방법을 구분하여 제시합니다.
- 수치 자료에는 출처 또는 계산 근거를 명시합니다.
- 주요 위험 요소는 관련 근거가 있는 경우 별도의 항목으로 나열합니다.
- 출력 결과는 결론을 먼저 제시하고, 마지막에는 실질적인 다음 단계를 제안합니다.
