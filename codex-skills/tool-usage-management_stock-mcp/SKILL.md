# Stock MCP Skill

# Must

## Scope
- StockSearcher MCP 도구를 사용하여 주식 목록, 가격 변동 내역, 예측 날짜, 예측 행 또는 UpperBand 스캐너 결과를 검색할 때 이 문서를 반드시 적용해야 합니다.
- 모든 주식 관련 질문에 하나의 도구를 재사용하는 대신, 정확한 데이터 요구 사항에 따라 도구를 선택해야 합니다.

## Source of Truth
- 이 문서는 `./SKILL.md`에 있는 시장/날짜 매개변수 검증, 주식 도구 선택 및 주식 쿼리 의도 보고에 대한 내용을 다룹니다. 공유 워크스페이스 중지 조건, 위임된 런타임 제어 또는 주식 분석 설명 구조에 대한 내용은 다루지 않습니다.
- `AGENTS`는 워크스페이스 실행 트리거 및 워크플로 중지 조건에 대한 유일한 기준 문서입니다. 시장/날짜 검증 또는 주식 도구 선택 규칙이 아닌, 현재 워크스페이스 워크플로 내에서 주식 MCP 작업이 진행될 수 있는지 여부를 결정할 때 `AGENTS`를 참조하십시오.
- `../runtime-management_work-runtime/SKILL.md` 파일은 에이전트에서 위임하는 공유 MCP 유효성 검사, 공유 중지 조건 및 공유 보고 제어에 대한 유일한 기준 문서입니다. MCP 실행 전에 적용되는 공통 런타임 검사를 결정할 때 이 파일을 참조하십시오. 주식별 매개변수 규칙을 결정할 때는 이 파일을 참조하지 마십시오.

## Tool Selection Rules
- 시장별 주식 코드 또는 이름을 검색하는 작업에는 `list_stocks`를 사용해야 합니다.
- 특정 주식 코드에 대한 과거 가격 또는 거래량 행이 필요한 작업에는 `stock_data`를 사용해야 합니다.
- 예측 마감일이 필요한 작업에는 `list_predict_dates`를 사용해야 합니다.
- 특정 시장과 특정 마감일에 대한 예측 행이 필요한 작업에는 `predict_rows`를 사용해야 합니다.

- UpperBand 상단 조건을 가격 및 거래대금 기준으로 스캔하는 작업에는 `scanner_rows`를 사용해야 합니다.

## Parameter Validation Rules
- 도구에서 `market` 매개변수를 허용하는 경우, 모든 주식 MCP 호출 전에 해당 매개변수의 유효성을 검사해야 합니다.
- 현재 MCP 동작에서 확인된 지원되는 시장 코드만 사용해야 합니다.
- `stock_data`를 호출하기 전에 `code`의 유효성을 검사해야 합니다.
- `predict_rows`를 호출하기 전에 `as_of`의 유효성을 검사해야 합니다.
- 날짜 매개변수가 제공되는 경우 날짜 문자열이 `YYYY-MM-DD` 형식인지 확인해야 합니다.

- `scanner_rows`를 호출하는 경우, MCP 스키마에서 실제로 확인된 매개변수만 사용해야 합니다.
- `scanner_rows`에서 확인되지 않은 신규 매개변수를 사용자가 요청하는 경우, 해당 매개변수는 `unverified`로 표시하고 현재 확인된 매개변수만으로 조회 가능한지 먼저 판단해야 합니다.

## Query Framing Rules
- 도구를 호출하기 전에 사용자가 탐색 데이터, 과거 데이터, 사용 가능한 예측 날짜 또는 예측 결과를 원하는지 명시해야 합니다.
- 작업이 둘 이상의 시장에 적용될 수 있는 경우 시장을 명시적으로 지정해야 합니다.
- 요청된 과거 범위가 모호해질 수 있는 경우 행 제한 또는 날짜 창을 명시해야 합니다.
- 추천 또는 후보군 선별 요청에서는 후보군 정리용 조회와 개별 종목 후속 분석용 조회를 구분해서 필요한 데이터를 먼저 설명해야 합니다.

## Scanner Query Rules
- 이 섹션은 `scanner_rows` 단독 조회 규칙과 추천형 요청에서의 제한적 도구 조합 예시를 함께 다룹니다.
- `scanner_rows`의 현재 확인된 매개변수 이름은 `market`, `scan_date`, `close_max`, `trans_amnt_min`입니다.
- 현재 세션에서 `scanner_rows(market='JP', close_max=1000, trans_amnt_min=500000000)` 호출이 성공했으므로, `scan_date`는 미지정 조회가 가능한 것으로 관측되었습니다. 다만 이 동작은 응답 기준으로 확인된 사항이므로 실제 반환된 날짜를 함께 보고해야 합니다.
- 스캐너 검색에서는 사용자가 요구한 가격 상한, 거래대금 하한, 기준 시장, 기준일을 각각 어느 매개변수에 대응하는지 명시해야 합니다.
- 사용자가 "60일 UpperBand 상단" 조건을 요청하는 경우, 응답 필드에 `upperband60_1`가 존재하는지 확인하고 현재 스캐너가 60일 기준 결과를 반환하는지 근거와 함께 보고해야 합니다.
- 사용자가 요청한 기간 또는 밴드 관련 신규 파라미터가 MCP 스키마에서 확인되지 않은 경우, 지원되는 것으로 단정해서는 안 됩니다.
- 예시: JP 시장에서 가격 1000 이하, 거래대금 500000000 이상 조건으로 UpperBand 상단 종목을 찾는 경우 `scanner_rows(market='JP', close_max=1000, trans_amnt_min=500000000)` 순서로 조회하고, 미확인 파라미터 사용 여부를 별도로 보고해야 합니다.
- 추천 요청에서 스캔 기반 후보군이 필요한 경우 `scanner_rows` 또는 `predict_rows`로 후보군을 만들고, 후속 검토가 필요한 종목에 한해 `stock_data`로 시계열을 확인해야 합니다.
- `predict_rows`는 후보군 보조 신호로 사용할 수 있지만 단독 근거로 추천 결론을 내리면 안 되며, 실제 가격·거래대금·추세 확인은 `stock_data` 또는 `scanner_rows` 결과와 함께 보고해야 합니다.
- 예시: "일본주 추천" 요청에서 기술적 후보군은 `scanner_rows(market='JP', scan_date='YYYY-MM-DD')`, 예측 보조 신호는 `predict_rows(as_of='YYYY-MM-DD', market='JP')`, 개별 확인은 `stock_data(market='JP', code='<candidate>')`로 나눠 사용할 수 있습니다.

## Output Rules
- 어떤 주식 MCP 도구가 사용되었는지, 그리고 그 이유를 보고해야 합니다.
- 원시 출력만 제공하는 대신 반환된 행을 일반적인 언어로 요약해야 합니다.
- 복수 종목 추천을 위한 조회에서는 어떤 도구가 후보군 정리, 시장 조사, 개별 후속 확인에 각각 사용되었는지 구분해서 보고해야 합니다.
- 지원되지 않는 시장 또는 필수 매개변수가 누락된 경우 누락된 필드를 정확하게 표시하여 차단됨으로 표시해야 합니다.

# Must NOT

- `market`과 `as_of` 매개변수 없이 `predict_rows`를 호출해서는 안 됩니다.
- 특정 시장 조건의 스캐너 검색에서 `market` 없이 `scanner_rows`를 호출해서는 안 됩니다.
- `market`과 `code` 매개변수 없이 `stock_data`를 호출해서는 안 됩니다.
- 현재 MCP 동작에서 확인된 시장 외에 지원되지 않는 시장을 가정해서는 안 됩니다.
- 예측 데이터를 가격 변동 이력 데이터처럼 사용해서는 안 됩니다.

# Flow
1. 요청이 데이터 검색, 가격 변동 이력 조회, 예측 날짜 조회, 예측 행 조회 또는 스캐너 기반 조건 검색인지 확인합니다.
2. 해당 주식 MCP 도구를 선택합니다.
3. `market` 및 해당 도구에 필요한 모든 매개변수를 검증합니다.
4. 도구 호출을 실행합니다.
5. 도구 선택, 반환 결과의 의미, 그리고 매개변수가 차단된 경우 그 이유를 보고합니다.

# Definition of Done

## Verification
- 선택한 주식 MCP 도구가 사용자의 데이터 요구 사항과 일치합니다.
- 실행 전에 필수 매개변수의 유효성을 검사합니다.
- 시장 지원은 근거에 기반합니다.
- 최종 출력에는 도구 선택 및 결과의 의미가 설명되어 있습니다.
