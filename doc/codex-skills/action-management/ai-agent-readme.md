# ai-agent 스킬 설명

## 목적
- AI 에이전트 설계 시 기능 나열이 아니라 운영 문제, 의사결정 경계, 툴 계약, 검증 계획을 먼저 고정하도록 만든 스킬이다.

## 핵심 규칙
- 문제 정의를 아키텍처보다 먼저 작성한다.
- 자동 실행, 승인 필요, 금지 행동을 분리한다.
- 파이프라인을 입력 수집부터 로깅까지 순서대로 정의한다.
- LLM 호출과 MCP/툴 연동은 입력·출력 계약, 실패 처리, 타임아웃까지 함께 적는다.
- 상태 저장, 세션, 캐시는 목적별로 분리한다.

## 사용 시 주의
- 구현 세부 규칙은 이 문서가 아니라 `coding-assistant.md`를 따른다.
- 프롬프트 문구만으로 안전성이나 정책 통제를 대신하면 안 된다.
- 실패 처리와 관측 가능성이 빠진 설계는 완료로 보면 안 된다.
