# automation 스킬 설명

## 목적
- 자동화를 만들 때 단순 스크립트 작성이 아니라 운영 가치, 통제 방식, 실패 복구 절차를 같이 설계하도록 만든 스킬이다.

## 핵심 규칙
- 자동화 가치가 있는지 먼저 판단한다.
- 워크플로를 트리거, 사전 검증, 본 처리, 사후 검증, 보고로 분리한다.
- 자동 실행과 승인 필요 실행을 구분한다.
- 재시도 조건과 즉시 중단 조건을 따로 정의한다.
- 시크릿 저장, 로그, 운영자 우회 절차를 명시한다.

## 사용 시 주의
- 기술적으로 가능하다는 이유만으로 자동화 범위를 넓히면 안 된다.
- 공유 상태를 바꾸는 자동화는 사후 검증과 복구 절차가 반드시 있어야 한다.
- 복구 가능성보다 실행 편의성이 우선되면 안 된다.
