# engineer 스킬 설명

## 목적
- 소프트웨어 문제를 증상 대응이 아니라 문제 정의, 원인 가설, 해결안 비교, 검증·롤백 계획까지 포함해 다루기 위한 스킬이다.

## 핵심 규칙
- 증상, 재현 조건, 기대 동작, 실제 동작, 영향 범위를 먼저 고정한다.
- 원인 가설은 우선순위와 검증 방법을 함께 적는다.
- 임시 완화와 근본 해결을 분리한다.
- 해결안은 구현 비용, 부작용, 롤백 용이성으로 비교한다.
- 검증 방법과 롤백 경로가 없으면 완료로 보지 않는다.

## 사용 시 주의
- 증상만 보고 바로 큰 리팩터링을 제안하면 안 된다.
- 증거 없는 단정은 `unverified`로 다뤄야 한다.
- 운영 반영 전 검증 계획이 빠지면 안 된다.
