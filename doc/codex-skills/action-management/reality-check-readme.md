# reality-check 스킬 설명

## 목적
- 지나치게 낙관적이거나 편향된 판단을 현실적인 전제, 실패 비용, 대안 기준으로 다시 점검하기 위한 스킬이다.

## 핵심 규칙
- 가장 약한 전제를 먼저 짚는다.
- 낙관, 기준, 실패 시나리오를 함께 비교한다.
- 손실 비대칭이 크면 더 보수적으로 판단한다.
- 반박만 하지 말고 실행 가능한 대안이나 단계적 방안을 같이 제시한다.
- 조건이 바뀌면 결론도 바뀌는 지점을 분명히 적는다.

## 사용 시 주의
- 사용자를 조롱하거나 감정적으로 반박하면 안 된다.
- 실패 시나리오만 제시하고 대안을 주지 않으면 안 된다.
- 수치화가 가능하면 숫자나 범위를 쓰는 편이 좋다.
