- Claude review runtime entry skill을 runtime-management로 이동했고, mandatory review phase와 `source-review` 단계를 반영했다.

# 원인

- `claude-review-runtime`은 skill governance 규칙보다 Claude CLI 실행 전 사전 확인과 runtime entry에 가까운 역할을 가져 runtime-management에 두는 편이 더 적절했다.

# 변경 내용

- 파일 위치를 `skill-management`에서 `runtime-management`로 이동했다.
- protocol 참조 경로를 새 위치 기준으로 유지했다.
- runtime entry wrapper 역할만 유지하고 protocol 중복 정의는 두지 않았다.
- review applicability를 optional이 아니라 mandatory 기본값으로 수정했다.
- `source-review` 단계를 review phase 목록에 추가했다.

# 재발 방지

- 실행 전 사전 확인과 runtime handoff를 담당하는 문서는 runtime-management 아래에서 관리한다.
- governance 문서와 runtime entry 문서를 역할 기준으로 분리한다.
- runtime review 수정 시 applicability, blocked 처리, phase 목록을 함께 점검한다.
