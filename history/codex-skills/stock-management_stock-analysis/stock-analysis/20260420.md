# 수정 요약

- `stock-analysis`에 추천형 분석의 기본 흐름인 후보군 선별, 시장 조사, 추천 분류, 보고서 순서를 추가했다.

# 원인

- 실제 추천 요청은 "후보군 먼저 선정 -> 시장조사 -> 추천 종목 선별 -> 공격형/보수형 분류" 순서를 요구했지만 기존 문서는 이 흐름을 부분 예시로만 다뤘다.
- 시장 조사에서 지수, 업종, 테마, 매출 증가나 실적 방향성 같은 기본적 맥락을 어떻게 다뤄야 하는지 최소 커버리지 규칙이 부족했다.

# 변경 파일

- `D:/work/nowonbun-harness/codex-skills/stock-management_stock-analysis/SKILL.md`

# 변경 내용

- `Minimum Analysis Coverage Rules`에 추천형 요청의 후보군 선별 선행, 시장 조사 최소 커버리지, 매출 증가·실적 방향성의 근거 요구, 보수형/중립형/공격형 분류 근거를 추가했다.
- `Example Analysis Procedure Rules`에 "후보군 먼저 뽑고 시장조사까지 해서 추천해줘" 예시를 추가했다.
- `Output Rules`에 추천 보고서의 기본 섹션 순서를 추가했다.
- `Flow`에 후보군 정리, 시장 조사 요약, 위험 성향별 분류 단계를 반영했다.

# 재발 방지

- 추천형 분석 규칙을 수정할 때는 후보군 정리, 시장 조사, 추천 분류, 출력 순서를 같은 변경 세트로 확인한다.
- 매출 증가나 실적 방향성, 테마를 언급할 때는 항상 근거를 확인하고 근거가 부족하면 `unverified`로 표시한다.
