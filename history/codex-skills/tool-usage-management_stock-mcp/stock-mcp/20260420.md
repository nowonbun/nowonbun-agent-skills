# 수정 요약

- `stock-mcp`에 추천형 요청에서 후보군 정리용 조회와 후속 개별 검토 조회를 구분하는 규칙을 추가했다.

# 원인

- 실제 추천 분석에서는 `scanner_rows`, `predict_rows`, `stock_data`를 역할별로 나눠 사용했지만 기존 문서는 각 도구의 개별 규칙 중심이라 추천형 조합 사용 이유를 설명하기 어려웠다.
- `predict_rows`가 후보군 보조 신호인지 실제 추천 근거인지 구분하는 문장이 부족했다.

# 변경 파일

- `D:/work/nowonbun-harness/codex-skills/tool-usage-management_stock-mcp/SKILL.md`

# 변경 내용

- `Query Framing Rules`에 추천 또는 후보군 요청에서 후보군 정리용 조회와 후속 분석용 조회를 구분해서 설명해야 한다는 규칙을 추가했다.
- `Scanner Query Rules`에 추천형 요청에서 `scanner_rows` 또는 `predict_rows`로 후보군을 만들고 `stock_data`로 후속 확인하는 예시를 추가했다.
- `predict_rows`는 후보군 보조 신호이며 단독 추천 근거로 사용하면 안 된다는 규칙을 추가했다.
- `Output Rules`에 후보군 정리, 시장 조사, 개별 후속 확인에 사용된 도구를 구분해 보고해야 한다는 규칙을 추가했다.

# 재발 방지

- 추천형 MCP 규칙은 도구 조합 예시 수준으로만 추가하고 개별 도구의 필수 매개변수 규칙과 중복 작성하지 않는다.
- 예측 데이터는 항상 후보군 보조 신호로 설명하고 실제 추천 결론은 시계열 또는 스캐너 결과와 함께 제시한다.
