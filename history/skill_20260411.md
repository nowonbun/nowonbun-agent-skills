# Purpose

- skill-management 규칙 문서의 언어 정책과 companion 문서 경로를 현재 프로젝트 기준으로 정렬한다.

# Incident Summary

- 기존 skill-management 규칙 문서가 다른 프로젝트 기준의 일본어/`harness/doc` 규칙을 유지하여 현재 프로젝트의 한국어/`doc` 기준과 불일치했다.

# Root Cause

- 과거 프로젝트에서 사용하던 규칙 문서를 현재 저장소로 가져오는 과정에서 저장소 루트 구조와 문서 언어 정책을 현재 프로젝트 기준으로 재정렬하지 않았다.

# Impact

- skill 문서 작성 시 한국어 기본 정책과 충돌할 수 있었다.
- companion 문서 생성 경로가 현재 저장소 구조와 맞지 않아 산출물 경로 검증이 실패할 수 있었다.
- 신규 history 파일 기본 언어가 일본어로 기록되어 현재 프로젝트 문서 기준과 어긋날 수 있었다.

# Recurrence Prevention Rules

- skill-management 규칙 문서를 다른 프로젝트에서 가져올 때 저장소 루트 기준 경로를 먼저 검증한다.
- skill-management 규칙 문서를 수정할 때 기본 문서 언어와 history 기본 언어가 현재 프로젝트의 상위 규범과 일치하는지 먼저 검토한다.
- companion 문서 경로 규칙을 바꿀 때 실제 저장소 폴더 존재 여부를 파일 시스템 증거로 확인한다.
- action-management 스킬을 수정할 때는 기존 설명형 문서를 그대로 부분 보수하지 말고 `# Must`, `# Must NOT`, `# Definition of Done` 구조 충족 여부부터 검사한다.
- skill 본문을 수정할 때는 같은 변경 세트에 companion readme와 skill-scoped history를 함께 생성한다.
- tool-usage-management 스킬은 실제 MCP 응답으로 확인된 권한·파라미터·시장 범위와 문서 규칙을 대조한 뒤 수정한다.
- cross-review 설계를 수정할 때는 applicability, phase 목록, blocked 처리, protocol 로그, action-management 호출 규칙을 함께 정렬한다.

# Checklist

- 기본 문서 언어 규칙이 현재 프로젝트와 일치하는가
- companion 문서 경로가 실제 저장소 구조와 일치하는가
- history 기본 언어 규칙이 현재 프로젝트와 일치하는가
- 수정한 규칙 문서 경로를 history에 정확히 기록했는가

# Conclusion

- `harness/doc` 경로 규칙을 `doc` 경로 규칙으로 수정했고, companion/history 기본 언어를 한국어로 맞췄다. 단, skill rule content 본문 언어는 영어 원칙을 유지하도록 정정했다.
- Claude cross-review 규칙은 현재 프로젝트의 `CLAUDE.md` 기준 정합성/데그레/리그레션 축에 맞게 재정렬했고, `claude-review-runtime.md`는 runtime-management 아래에서 프로토콜을 호출하는 얇은 실행 래퍼로 재배치했다.
- runtime governance와 cross-review protocol 사이의 경로 및 source-of-truth 연결을 현재 프로젝트 기준으로 다시 연결했다.
- action-management 하위 6개 스킬은 한국어 설명형 문서에서 영문 실행 규칙 문서로 재작성했고, 각 스킬별 한국어 readme와 history 파일을 함께 추가했다.
- tool-usage-management 하위 `github-mcp.md`, `mariadb-mcp.md`, `stock-mcp.md`도 같은 형식으로 재작성했고, 현재 MariaDB/Stock MCP 응답 기준으로 권한과 파라미터 규칙을 정렬했다.
- cross-review 관련 스킬은 optional 설계에서 mandatory 설계로 전환했고, 리뷰 범위를 `plan-review`, `source-review`, `result-review`, `re-review`로 재정렬했다.
