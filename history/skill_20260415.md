## Additional Incident - codex-skills path standard

### Purpose

- Record why and how the active skill source layout was changed to `<category>_<skill>/SKILL.md`.

### Incident Summary

- When skill sources used `<category>/<skill>.md`, deployment copied them into `.codex/skills/<category>_<skill>/SKILL.md`, so the source layout and deployed layout diverged and relative-path assumptions could drift.

### Root Cause

- Source layout and deployed layout were different.
- Install scripts could normalize `.../SKILL.md` into an incorrect `..._SKILL` name unless they explicitly use the parent directory name.
- Skill-scoped history paths were anchored to the old relative layout, so moving only source files would split future history accumulation.

### Impact

- Relative-path assumptions could differ between source and deployed skill locations.
- Install scripts could generate wrong skill names for the canonical source layout.
- Skill-scoped history continuity could break unless the history directories moved together.

- `codex-skills`와 `D:/work/AGENTS.md`에서 스킬명으로 적힌 참조를 상대 `SKILL.md` 경로로 정규화한 이유와 결과를 기록한다.

- 여러 스킬 문서가 다른 스킬을 `skill-modify-history`, `claude-review-runtime` 같은 스킬명으로 참조하고 있었고, 사용자는 이를 각 문서 기준 상대 `.md` 경로로 바꾸길 요청했다.
