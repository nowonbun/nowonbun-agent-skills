# Purpose
- 코드 수정 작업에서 계획 수립과 Claude 교차 검토가 누락된 재발 원인을 기록하고, 스킬 규칙을 강화하여 같은 누락이 다시 발생하지 않도록 한다.

# Incident Summary
- `D:/work/StockSearcher/webapp.py` 수정 작업에서 실행 가능한 계획을 사용자에게 제시했지만, 해당 계획에 대한 `plan-review`를 실행하지 않은 채 소스 수정을 진행했다.
- 같은 작업에서 소스 수정 후 필수 `source-review`와 완료 전 `result-review`도 수행하지 않았다.
- 이후 사용자 지적으로 누락이 확인되어 관련 스킬 규칙을 강화해야 하는 상태가 발생했다.

# Root Cause
- 코드 변경 작업에서 `coding-assistant`의 Claude review 규칙을 확인했지만, 실제 쓰기 실행 직전에 `plan-review` 완료 여부를 차단 조건으로 강제하지 못했다.
- `claude-review-runtime`에는 실행 가능한 계획 부재 차단은 있었지만, 계획이 존재해도 `plan-review`가 아직 실행·보고되지 않은 상태를 별도 차단하는 문구가 부족했다.
- 실행 체크리스트보다 구현 진행이 앞서면서 필수 검토 단계를 완료 상태로 재확인하는 절차가 빠졌다.

# Impact
- `stock-analysis`에 실제 분석 절차 예시 섹션을 추가해 후보군 선별형 분석, 단일 종목 심화 분석, 예측 데이터 포함 분석의 진행 예시를 문서화했다.
- source-review에서 scanner_rows 관련 Scope·Flow·Must NOT 정합성과 scan_date 설명, 그리고 stock-analysis의 용어 모호성을 추가 수정 대상으로 확인했다.
- 코드 변경이 필수 교차 검토 없이 적용되어 검토 절차 신뢰성이 낮아졌다.
- 사용자에게 보고되기 전까지 `plan-review`, `source-review`, `result-review` 누락이 감지되지 않았다.
- 향후 유사 작업에서도 같은 누락이 반복될 위험이 확인되었다.

- source-review에서 scanner_rows 관련 Scope·Flow·Must NOT 정합성과 scan_date 설명, 그리고 stock-analysis의 용어 모호성을 추가 수정 대상으로 확인했다.
- 사용자가 스캐너 기반 후보군 검색을 요청할 때 응답자가 도구 선택과 파라미터 검증 근거를 별도로 재구성해야 했다.

- 사용자 요청은 "후보군 먼저 선정, 시장조사, 추천 종목 선별, 공격형/보수형 분류" 순서를 요구했으나 기존 스킬 문서에는 이 흐름이 부분적으로만 반영되어 있었다.

# Impact
